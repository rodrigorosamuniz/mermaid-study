# Security Policy

These repositories are public educational labs. Do not use real personal data, production credentials, private keys, API tokens, or confidential company data in issues, pull requests, examples, screenshots, or test payloads.

## Reporting a Vulnerability

If you find a vulnerability or accidentally exposed secret, do not open a public issue. Use GitHub private vulnerability reporting from the Security tab when available. If that is not available, contact the repository owner through GitHub before sharing technical details publicly.

## Pull Requests From Forks

Pull requests from external contributors may require manual approval before GitHub Actions workflows run. Workflow changes, scripts, dependency updates, and generated files should be reviewed carefully before approval.

## Supported Scope

These projects are classroom labs, not production systems. Security fixes that reduce risk for students, instructors, or public users are welcome.
